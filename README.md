# VISION Admin Portfolio

Django 기반 관리자 페이지를 중심으로, 웹 접속 보안 시스템의 정책 관리와 이력 조회, 로그 모니터링 기능을 구현한 팀 프로젝트 포트폴리오입니다.

## 프로젝트 소개
이 프로젝트는 사용자의 웹 접속 요청을 분석하여 유해 사이트 접속을 차단하고, 관리자 페이지에서 정책을 등록·수정·관리할 수 있도록 구성된 웹 접속 보안 관리 시스템입니다.

탐지 엔진이 1차적으로 요청을 검사하고, 관리자 페이지에서는 정책 관리 및 로그 조회 기능을 제공합니다. 또한 FastAPI 기반 AI 분석 서버와 연동하여 미탐지 요청에 대한 2차 분석 결과를 기록하고 검토할 수 있도록 구성했습니다.

## 담당 역할
- Django 관리자 페이지 구현
- 정책 조회 / 추가 / 수정 / 삭제 기능 구현
- 정책 수정 이력 / 삭제 이력 조회 기능 구현
- AI 분석 이력 및 로그 관리 화면 구현
- 관리자 페이지와 탐지 엔진 간 정책 반영 흐름 확인
- 운영 흐름을 고려한 데이터 구조 및 이력 관리 방식 개선

## 주요 기능
### 1. 정책 관리
- 도메인 / 정규표현식 기반 정책 등록
- 적용 여부 및 처리 유형(block / log) 관리
- 정책 상세 조회 및 수정 기능 제공

### 2. 정책 이력 관리
- 정책 수정 내역 조회
- 변경된 항목 비교 기능 제공
- 삭제된 정책 이력 관리 및 복구 고려 구조 반영

### 3. AI 분석 이력 관리
- FastAPI 분석 결과 조회
- 관리자 검토 여부 및 처리 결과 반영
- 정책 추가 및 무시 처리 흐름 제공

### 4. 로그 모니터링
- 탐지 로그 조회 및 검색 필터 제공
- URL, 정책명, IP, 정책 타입, 시간 조건 기반 검색 지원

## 시스템 구성
- **탐지 엔진**: 웹 접속 요청을 1차적으로 검사하고 정책에 따라 차단 또는 기록 수행
- **FastAPI**: 1차 정책에서 처리되지 않은 요청에 대해 2차 AI 분석 수행
- **Django 관리자 페이지**: 정책 관리, 로그 조회, AI 분석 결과 검토 기능 제공

## 동작 흐름
1. 사용자의 웹 접속 요청이 탐지 엔진으로 전달됩니다.
2. 탐지 엔진은 정책 기준으로 요청을 검사합니다.
3. 차단 대상이면 즉시 차단하고 기록을 남깁니다.
4. 정책에 걸리지 않은 요청은 FastAPI로 전달되어 AI 분석이 진행됩니다.
5. 관리자는 Django 관리자 페이지에서 정책, 이력, 로그, AI 분석 결과를 조회하고 관리할 수 있습니다.

## 문제 해결 경험
### 정책 삭제 이력 구조 개선
프로젝트 초기에는 정책 ID를 기준으로 구조를 단순하게 연결하는 방식을 고려했습니다. 그러나 삭제 처리와 이력 관리가 함께 이어지는 과정에서 제약이 발생했고, 운영 흐름까지 고려한 구조로 다시 정리할 필요가 있었습니다.

이후에는 단순히 연결 관계만 보는 것이 아니라, 수정·삭제·이력 관리가 실제로 어떤 방식으로 이어지는지 함께 검토하며 데이터 구조를 조정했습니다. 이 경험을 통해 기능 단위가 아니라 운영 과정까지 고려하는 방식으로 시야를 넓힐 수 있었습니다.
